子供のきもち - AI Rules

このファイルは AIがこのプロジェクトのコードを変更・生成する際に守るべきルールを定義する。
AIは以下のルールを必ず優先すること。

1. 最重要原則

このアプリは 感情記録アプリではない。

これは

子どもの感情が高ぶった瞬間に行動を提示するアプリ

である。

したがって AI は次を守ること。

子ども向けUIを最優先する

行動提示を中心体験とする

親向け情報は補助とする

UIを複雑にしない

2. UI設計ルール
子ども向け画面

子ども向けUIでは必ず次を守る。

ルール

表示情報を増やしすぎない

文字量を増やしすぎない

行動は 1つだけ提示する

抽象説明より具体行動を優先する

禁止例

行動を複数並べる

長い説明を書く

選択肢を増やす

親向けUI

親向け情報は補助機能である。

ルール

初期状態では開かない

アコーディオンで隠す

子ども向け表示を邪魔しない

3. UXルール

AIは次のUX思想を壊してはいけない。

原則

子どもが理解できる

操作が少ない

成功体験を強く返す

親向け情報は二段階目

4. 成功演出ルール

成功演出はこのアプリの価値の一部である。

AIは以下を削除してはいけない。

褒め言葉表示

成功カード

紙吹雪アニメーション

これらは 感情調整の成功体験を作るための重要要素である。

5. パフォーマンスルール

AIはアプリの軽さを維持すること。

必須ルール

重いコンポーネントは lazy import

グラフは必要なときのみ読み込む

起動時にレポート処理を走らせない

対象

EmotionReportCharts

6. データ構造ルール

AIは既存のデータ構造を壊さないこと。

Emotion
type Emotion = {
  id: string
  label: string
  image: ImageSource
  bgColor: string
  empathyMain: string
  empathy: string[]
  actions?: string[]
  parentActions: string[]
  parentGuide: string
  evidenceNote?: string
}

ルール

新しいフィールド追加は慎重に行う

UIとデータ構造の整合性を崩さない

7. 日付処理ルール

このプロジェクトでは

ローカル日付基準

で処理する。

禁止

AIは以下を直接使ってはいけない。

toISOString()

理由

タイムゾーンで日付がズレるため。

代わりに以下を使う。

getLocalDateKey()
getLocalDateKeyFromDate()
8. 履歴画面ルール

履歴画面は単なるログではない。

役割

振り返り

プレミアム価値

レポート導線

AIは次を守ること。

表示ルール

今日 → 常に展開

最近 → 折りたたみ

古い履歴 → 集約表示

9. レポート機能ルール

レポートは プレミアム機能 である。

必須ルール

PremiumGate の外に出さない

未課金ユーザーでグラフを読み込まない

lazy import を維持する

10. コード構造ルール

AIは次の責務分離を守る。

app/

画面

components/

再利用UI

constants/

静的データ

lib/

ロジック

11. AIが変更してはいけない思想

AIは次の思想を崩してはいけない。

最重要

子ども向けUIを最優先

次

親向け情報は補助

UX

行動は 1つだけ提示

12. 新機能追加ルール

AIが新機能を追加する場合は次を守る。

必須

子ども向け画面を複雑にしない

親向け機能として追加する

パフォーマンスを悪化させない

13. AI向け最終要約

このアプリは

子どもの感情が高ぶった瞬間に

共感

行動提示

成功体験

を提供するアプリである。

最重要UXは

「いまやること」を1つ提示すること

である。

AIはこの思想を崩してはいけない。
